DESPLIEGUE DE APLICACIONES

- git init: crea subdirectorio .git que contiene el esqueleto del respositorio de Git para empezar a desarrollar.

- git fetch: descarga cambios realizados en el repositorio remoto
- git merge: aplica los cambios descargados a la rama en la que nos encontramos
- git pull: fetch+merge

- git commit: Confirma los cambios realizados (git commit -m "mensaje")

- git push origin <nombrerama>: Sube la rama al servidor remoto

- git status: muestra el estado actual de la rama, así como cambios sin commitear

- git add: NO es subir; comienza a seguir el archivo

- git checkout -b <nombreramanueva>: Creamos nueva rama y nos metemos en ella
- git checkout -t origin/<nombrerama>: Volvemos a rama anterior guardando rama en local

- git branch: Muestra todas las ramas locales
- git branch -a: Muestra todas las ramas locales y remotas

DESPLIEGUE DE APLICACIONES:

Los componentes de un equipo informático debe tener entrada, salida o ambas.

CUIDADO entre bits y Bytes y su equivalencia inexacta (1 Byte = 8 bits)

Redes:

por *Area de Distribucion*
- PAN -> Red de Area Personal (Personal Area Network) pequeñita. Bluetooth.
- LAN -> Red de Area Local (Local Area Network) tamaño peque-mediano. Casa.
- MAN -> Red de Area Metropolitana (Metropolitan...) red de alta velocidad (banda ancha) que da cobertura a un área geográfica, como la fibra óptica de una ciudad.
- WAN -> Red de Area Amplia (Wide Area Network) intercontinental, por satélite. 

por *Relación Funcional*
- Cliente-Servidor
- Peer-to-Peer(P2P)

por *Tecnología*
- Point-to-Point: Parejas individuales de máquinas. Un switch.
- Broadcast: Un canal para varias máquinas. 

por *Topología física* (cableado)
- Bus, troncal o backbone...
